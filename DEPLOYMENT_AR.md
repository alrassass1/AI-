# تعليمات النشر على استضافة نيم شيب (Namecheap)

بما أن استضافة نيم شيب المشتركة (Shared Hosting) تعتمد بشكل أساسي على الملفات الثابتة (Static Files)، إليك الطريقة الصحيحة لتشغيل مشروع "رؤية" عليها:

## الخيار الأول: النشر كوقع ثابت (Static SPA) - الأسهل والأسرع
هذا الخيار سيجعل الموقع يعمل بالكامل (الذكاء الاصطناعي، الفحص، الواجهات) باستثناء نموذج الاتصال الذي يتطلب خادماً.

1. قم بتشغيل الأمر التالي في جهازك لبناء المشروع:
   ```bash
   npm run build
   ```
2. سيظهر لك مجلد جديد باسم `dist`.
3. ارفع **محتويات** مجلد `dist` إلى المجلد الرئيسي في الاستضافة (غالباً `public_html` أو مجلد الفرعي الخاص بـ `roaya-ai.nrh-med.com`).
4. تأكد من وجود ملف `.htaccess` (مرفق في المشروع) في نفس المجلد لضمان عمل الروابط بشكل صحيح.

## الخيار الثاني: تشغيل Node.js على نيم شيب
إذا كنت ترغب في تشغيل الخادم (server.ts) لتفعيل نموذج الاتصال البرمجي:
1. ابحث عن "Setup Node.js App" في لوحة تحكم cPanel.
2. اختر إصدار Node.js (يفضل 18 أو أحدث).
3. حدد مسار التطبيق (Application root).
4. حدد ملف التشغيل (Application startup file) ليكون `server.js` (بعد تحويله من ts).
5. قم برفع كافة ملفات المشروع وتشغيل `npm install` من داخل لوحة التحكم.

## نصيحة تقنية
بما أنك جربت **Vercel** وكان يعمل، فنحن ننصح بشدة باستخدام **Vercel** أو **Netlify** لنشر هذا النوع من المشاريع (Vite + React + Node.js) لأنها توفر دعماً تلقائياً ومجانياً لهذه التقنيات دون الحاجة لإعدادات معقدة.
